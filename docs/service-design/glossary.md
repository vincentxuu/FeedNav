# 術語表（Glossary）

> 本文件定義 FeedNav 服務設計文件中使用的標準術語，確保團隊溝通一致性。

---

## 一、產品功能術語

| 標準術語 | 定義 | 避免使用 |
|---------|------|---------|
| **收藏** | 用戶將餐廳加入個人清單的動作 | 儲存、加入最愛、bookmark |
| **口袋名單** | 用戶收藏的餐廳清單（面向用戶的說法） | 收藏列表、我的收藏 |
| **去過** | 用戶標記已造訪餐廳的動作 | 造訪、visited、打卡 |
| **美食足跡** | 用戶去過的餐廳紀錄（面向用戶的說法） | 造訪紀錄、歷史紀錄 |
| **隨機推薦** | 系統隨機挑選餐廳的功能 | 隨機探索、幫我決定 |
| **探索** | 用戶瀏覽、搜尋餐廳的行為 | 搜尋、查找 |

## 二、用戶相關術語

| 標準術語 | 定義 | 避免使用 |
|---------|------|---------|
| **用戶** | 使用 FeedNav 的人 | 使用者、客戶、會員 |
| **種子用戶** | 早期招募的測試用戶 | Beta 用戶、測試者 |
| **活躍用戶** | 7 天內有使用行為的用戶 | 常客、忠實用戶 |
| **註冊** | 建立 FeedNav 帳號的動作 | 加入、會員申請 |
| **登入** | 以現有帳號進入系統 | 簽入、sign in |

## 三、餐廳相關術語

| 標準術語 | 定義 | 避免使用 |
|---------|------|---------|
| **餐廳** | 提供餐飲服務的店家 | 店家、商家、美食 |
| **料理類型** | 餐廳的菜系分類（如日式、台式） | 菜系、類別、分類 |
| **行政區** | 台北市的行政區劃分 | 地區、區域 |
| **情境標籤** | 描述適用場景的標籤（如聚餐適合） | 場景、情境、用途 |

## 四、指標術語

| 標準術語 | 定義 | 計算方式 |
|---------|------|---------|
| **收藏數** | 餐廳被收藏的總次數 | COUNT(user_favorites) |
| **回訪率** | N 天內再次使用的用戶比例 | 回訪用戶 / 總用戶 |
| **收藏轉換率** | 訪問後有收藏行為的比例 | 有收藏用戶 / 訪問用戶 |
| **首次收藏率** | 註冊後 24 小時內收藏的比例 | 有首次收藏 / 新註冊 |
| **DAU** | 每日活躍用戶數 | 當日有行為的不重複用戶 |
| **MAU** | 每月活躍用戶數 | 當月有行為的不重複用戶 |
| **NPS** | 淨推薦分數 | 推薦者% - 批評者% |

## 六、文案風格指南

### 面向用戶的文案

| 場景 | 標準用法 | 避免用法 |
|------|---------|---------|
| 收藏按鈕 | 收藏 | 加入收藏、儲存 |
| 收藏頁標題 | 我的口袋名單 | 我的收藏、收藏列表 |
| 去過按鈕 | 去過 | 造訪、已造訪 |
| 去過頁標題 | 美食足跡 | 造訪紀錄、歷史 |
| 隨機按鈕 | 今天吃什麼？ | 隨機、隨機推薦 |
| 空狀態 | 你的口袋名單即將誕生 | 還沒有收藏 |

### 語氣原則

1. **輕鬆親切**：避免過於正式的用語
2. **行動導向**：強調「做什麼」而非「是什麼」
3. **正向鼓勵**：空狀態也要有期待感
4. **簡潔明瞭**：一句話說清楚
