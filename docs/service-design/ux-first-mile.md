# 用戶首次體驗設計（First Mile UX）

> **核心問題**：用戶覺得「只是吃個飯，不需要特別記錄」，所以不使用收藏功能
> **設計目標**：降低心理門檻，讓每個人都覺得記錄美食探索是值得的

---

## 一、問題診斷

### 現況分析

```
用戶行為矛盾：
┌─────────────────────────────────────────┐
│  愛找餐廳推薦  ≠  自己卻不記錄收藏      │
└─────────────────────────────────────────┘

心理機制：
「只是吃個飯而已」
「用 Google Maps 就好了」
「記錄太麻煩了」
「我又不是美食部落客」
```

### 這不是功能問題，是心理門檻

| 錯誤假設 | 實際問題 |
|---------|---------|
| 用戶不知道有收藏功能 | 用戶知道，但覺得沒必要 |
| 功能太複雜 | 功能不複雜，但「開始」很難 |
| 缺乏誘因 | 誘因不是問題，感受價值才是 |

---

## 二、設計原則

### 原則 1：重新定義「值得記錄」

**From**：只有美食家才需要記錄餐廳
**To**：每個人的美食探索都值得被記住

**設計手法**：
- 強調「發現」而非「評論」
- 展示「個人口袋名單」的價值
- 使用輕鬆的語言

### 原則 2：從最小承諾開始

**From**：請填寫詳細的餐廳評價
**To**：一鍵收藏，之後再說

**設計手法**：
- 漸進式揭露（Progressive Disclosure）
- 降低首次行動的成本
- 讓用戶先「嚐到甜頭」

### 原則 3：即時正向回饋

**From**：收藏了然後呢？（被動）
**To**：收藏後立刻看到價值（主動）

**設計手法**：
- 即時視覺化回饋
- 明確的進度感
- 展示收藏地圖

## 五、降低「沒必要」門檻的具體策略

### 策略 1：展示統計數據，讓用戶知道價值

```
「已有 1,234 位台北人在 FeedNav 建立了口袋名單」
「平均每人收藏 15 家餐廳」
「本週最多人收藏的餐廳是...」
```

### 策略 2：使用「發現」的語言，而非「記錄」

```
❌ 「記錄你吃過的餐廳」
✓ 「發現新的美食」

❌ 「建立你的餐廳資料庫」
✓ 「打造你的口袋名單」

❌ 「追蹤你的飲食紀錄」
✓ 「探索台北美食地圖」
```

### 策略 3：強調輕量化操作

```
┌─────────────────────────────────────────────────────────────┐
│  收藏只需要一秒鐘 ⚡                                         │
│                                                             │
│  看到喜歡的餐廳？                                            │
│  點一下 ❤️ 就收藏了                                          │
│                                                             │
│  不需要：                                                   │
│  ✗ 寫評論                                                  │
│  ✗ 上傳照片                                                │
│  ✗ 填寫任何資料                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 策略 4：提供「快速動作」降低門檻

```
┌─────────────────────────────────────────────────────────────┐
│  @餐廳詳情頁                                                 │
│                                                             │
│  快速動作：                                                  │
│  ┌────────┬────────┬────────┬────────┐                     │
│  │ ❤️     │ ✓      │ 📍     │ 📤     │                     │
│  │ 收藏   │ 去過   │ 導航   │ 分享   │                     │
│  └────────┴────────┴────────┴────────┘                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 六、空狀態設計（Empty State）

當頁面沒有內容時，不要讓用戶看到「空蕩蕩」，而是看到「機會」。

### 6.1 收藏頁空狀態

**不要**：
```
「你還沒有收藏任何餐廳」
「開始收藏餐廳」
```

**要**：
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  你的口袋名單即將誕生 ✨                                     │
│                                                             │
│  收藏喜歡的餐廳，下次不知道吃什麼                            │
│  打開這裡就有答案。                                          │
│                                                             │
│  [ 探索附近餐廳 ]                                           │
│                                                             │
│  💡 平均每人收藏 15 家餐廳                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 造訪紀錄空狀態

**不要**：
```
「還沒有造訪紀錄」
```

**要**：
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  你的美食足跡 👣                                             │
│                                                             │
│  去過的餐廳會自動出現在這裡，                                │
│  記錄你的台北美食探險。                                      │
│                                                             │
│  [ 標記去過的餐廳 ]                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 七、文案語氣指南

### 避免使用

| 避免 | 原因 |
|------|------|
| 「建立你的餐廳資料庫」 | 太正式，像在工作 |
| 「追蹤你的飲食」 | 暗示需要認真記錄 |
| 「分享你的美食評論」 | 排斥不愛寫的人 |
| 「成為美食達人」 | 過度競技化 |

### 建議使用

| 建議 | 原因 |
|------|------|
| 「收藏這家」 | 簡單、直接 |
| 「發現新餐廳」 | 強調探索樂趣 |
| 「你的口袋名單」 | 私密、個人化 |
| 「下次吃這家」 | 實用、生活化 |

## 八、成功指標

| 指標 | 定義 | 目標 |
|------|------|------|
| 註冊完成率 | 開始註冊 → 完成註冊 | > 70% |
| 首次收藏率 | 註冊後 24 小時內收藏 | > 50% |
| 首週收藏數 | 註冊後 7 天內收藏餐廳數 | > 5 |
| 7 日回訪率 | 註冊後 7 天內再次訪問 | > 30% |
| 收藏轉造訪率 | 收藏後實際標記去過 | > 20% |

## 九、總結

### 核心轉變

```
From: 「請記錄你的餐廳」（任務導向）
To:   「發現你的下一餐」（探索導向）

From: 「建立餐廳資料」（功能門檻）
To:   「一鍵收藏」（輕量化）

From: 「空白頁面等你填」（壓力）
To:   「推薦餐廳等你探索」（驚喜）
```

### 設計核心

**讓每個用戶都覺得：「這比 Google Maps 好用」**
