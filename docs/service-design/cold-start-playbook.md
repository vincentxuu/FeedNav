## 二、種子用戶策略（Phase 0）

### 2.1 種子用戶定義

**不是找「最愛吃的人」，是找「最願意嘗試新工具的人」**

```
理想種子用戶畫像：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  ✓ 常外食（每週 3+ 次）                                     │
│  ✓ 對新工具有興趣（早期採用者）                             │
│  ✓ 會分享好東西（有社群影響力）                             │
│  ✓ 住在台北、常在台北活動                                   │
│                                                             │
│  不一定是美食部落客，但一定是會分享的人                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 種子用戶數量目標

```
第一批：20-30 人
├─ 10 人：積極使用收藏功能（收藏 10+ 家）
├─ 10 人：標記造訪紀錄（去過 5+ 家）
└─ 10 人：願意分享到社群（FB/LINE）

為什麼是 20-30 人？
→ 足夠產生真實的使用數據
→ 可以收集有價值的反饋
→ 你可以一個一個邀請、一個一個輔導
```

### 2.3 邀請話術

**錯誤示範**：
```
「欸我做了一個找餐廳的網站，你來註冊一下」
→ 用戶會想：用 Google Maps 就好了
```

**正確示範**：
```
「我在做一個專門解決『正餐吃什麼』的平台。

 不管你是要聚餐找適合的店、一個人吃想找舒適的、
 還是在飲控想找健康選項，都能快速找到。
 還有隨機推薦功能，選擇困難直接幫你決定。

 想邀請你當第一批用戶，幫忙試用給點反饋。
 你願意幫忙嗎？」
```

**要點**：
- 說明「比 XX 好在哪」→ 建立價值
- 強調「解決什麼問題」→ 引起共鳴
- 問「願意幫忙嗎」→ 利用互惠心理

### 2.4 種子用戶輔導

**不要只是邀請，要輔導到活躍使用**

```
Day 1：發送邀請
└─ 「你願意幫忙嗎？」

Day 2：確認註冊
└─ 「註冊順利嗎？有任何問題可以問我」

Day 3：確認使用
└─ 「收藏幾家餐廳了嗎？隨機推薦功能試過嗎？」
└─ 如果還沒用：「讓我介紹幾個好用的功能」

Day 5：請求反饋
└─ 「有什麼覺得可以改進的地方嗎？」

Day 7：請求分享
└─ 「覺得好用的話，可以分享給朋友嗎？」
```

## 三、內容種子策略

### 3.1 確保「餐廳資料夠好」

**最大的冷啟動優勢**：資料品質

```
策略：確保餐廳資訊完整、準確、有用

資料品質檢查：
✓ 餐廳基本資訊正確（名稱、地址、電話）
✓ 評分資訊準確（來自 Google）
✓ 料理分類正確（AI 分類 + 人工驗證）
✓ 標籤有用（環境、服務、特色）
✓ 捷運站距離準確
```

### 3.2 熱門餐廳策展

**首頁要展示「最好的內容」**

```
策展標準：
✓ 評分 4.0+
✓ 有特色標籤
✓ 分布各行政區
✓ 涵蓋各料理類型

首頁展示策略：
┌─────────────────────────────────────────────────────────────┐
│  首頁 Banner                                                 │
│  「發現台北隱藏版美食」                                       │
│                                                             │
│  精選餐廳 1：高評分日式料理（品質保證）                       │
│  精選餐廳 2：特色小店（驚喜感）                               │
│  精選餐廳 3：平價美食（親民感）                               │
│                                                             │
│  「XX 人已經收藏了他們的口袋名單」                            │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 地區覆蓋策略

## 七、常見問題與對策

### Q1：種子用戶不活躍怎麼辦？

```
原因診斷：
├─ 不知道功能價值 → 主動介紹功能
├─ 沒時間 → 強調「只要收藏就好」
├─ 覺得沒必要 → 展示使用案例
└─ 忘記了 → 定期提醒

對策：
1. 主動演示好用的功能
2. 分享其他用戶的使用方式
3. 建立使用情境（週末找餐廳時打開 FeedNav）
```

### Q2：用戶來了但不註冊怎麼辦？

```
原因診斷：
├─ 不知道註冊有什麼好處 → 加強引導
├─ 註冊太麻煩 → 簡化流程
├─ 沒有立即需求 → 製造需求
└─ 懷疑價值 → 展示社會證明

對策：
1. 限制未登入功能（收藏需要登入）
2. 展示「XX 人已經收藏」
3. 提供 Google 一鍵登入
```

### Q3：內容不夠豐富怎麼辦？

```
對策：
1. 確保核心區域資料完整
2. 優先完善熱門餐廳資料
3. AI 自動補充分類和標籤
4. 用戶回報機制收集資訊
```

### Q4：成長停滯怎麼辦？

```
診斷方向：
├─ 獲客問題：沒有新用戶進來
├─ 留存問題：用戶來了就走
├─ 活躍問題：用戶不回來
└─ 價值問題：用戶覺得沒用

對策：
1. 回到用戶訪談，了解真正原因
2. 檢視每個環節的轉換率
3. A/B 測試新功能
4. 聚焦解決一個問題
```

## 九、總結

### 冷啟動成功的關鍵

```
1. 先有價值，再有用戶
   → FeedNav 的價值是「比 Google Maps 更好找餐廳」

2. 種子用戶要精心培養
   → 不是數量，是質量和活躍度

3. 降低門檻比增加功能重要
   → 「一鍵收藏」比「寫評論」優先

4. 社會證明很重要
   → 「XX 人已經收藏」比「歡迎註冊」有效

5. 口碑 > 廣告
   → 一個人真心推薦 > 十次官方發文
```

### 執行心態

```
冷啟動不是一次性的事，是持續的努力。

第一個月：手工做事（一個一個邀請、一個一個追蹤）
第二個月：找到模式（什麼有效、什麼沒用）
第三個月：開始規模化（用系統取代手工）

記住：每一個真正活躍的用戶，都是靠你親自邀請、親自跟進換來的。
```
