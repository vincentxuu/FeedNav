## 二、主要競品詳細分析

### 2.1 Google Maps

**市場地位**：最大通用地圖，餐廳搜尋的預設選擇

| 維度 | 分析 |
|------|------|
| **優勢** | 資料完整、用戶基數大、已安裝在所有手機 |
| **劣勢** | 不專為餐廳優化、篩選功能有限、收藏管理弱 |
| **收藏功能** | 有，但只是單純清單，無地圖視覺化 |
| **隨機推薦** | 無 |
| **情境篩選** | 無（只有基本分類） |

**用戶痛點**：
- 收藏了很多餐廳但難以管理
- 搜尋結果太雜，商業推廣混在其中
- 無法按「聚餐」「獨食」等情境篩選

**FeedNav 機會**：
- 提供更好的收藏管理（地圖視覺化）
- 專注餐廳，篩選更精準
- 情境標籤解決特定需求

---

### 2.2 愛食記

**市場地位**：台灣最大美食部落格聚合平台

| 維度 | 分析 |
|------|------|
| **優勢** | 內容豐富、SEO 強、部落客生態系 |
| **劣勢** | 內容品質參差、商業置入明顯、搜尋體驗差 |
| **收藏功能** | 有，但不是核心功能 |
| **隨機推薦** | 無 |
| **地圖探索** | 有，但體驗一般 |

**用戶痛點**：
- 很多文章是業配，難以信任
- 資訊過時（部落格文章可能是幾年前）
- 想快速找餐廳，卻要看長篇文章

**FeedNav 機會**：
- 提供即時、結構化的餐廳資訊
- 不依賴 UGC，資料品質可控
- 快速決策而非閱讀長文

---

### 2.3 窩客島

**市場地位**：美食評論社群平台

| 維度 | 分析 |
|------|------|
| **優勢** | 評論品質較高、社群氛圍好 |
| **劣勢** | 用戶規模較小、內容更新慢 |
| **收藏功能** | 有 |
| **隨機推薦** | 無 |
| **特色功能** | 美食活動、部落客合作 |

**用戶痛點**：
- 內容集中在特定類型餐廳
- 更新頻率不高

---

### 2.4 Foodpanda / UberEats

**市場地位**：外送平台，但也有探索功能

| 維度 | 分析 |
|------|------|
| **優勢** | 用戶基數大、可直接訂餐、優惠券吸引 |
| **劣勢** | 只有合作餐廳、偏向外送而非內用 |
| **收藏功能** | 有，但服務外送場景 |
| **推薦功能** | 有，基於訂餐歷史 |

**用戶痛點**：
- 想找內用餐廳但資料不完整
- 合作餐廳有限，無法涵蓋所有選擇

**FeedNav 機會**：
- 專注內用場景
- 涵蓋所有餐廳，不只合作店家

---

### 2.5 inline / EZTABLE

**市場地位**：訂位平台

| 維度 | 分析 |
|------|------|
| **優勢** | 解決訂位痛點、即時座位資訊 |
| **劣勢** | 只有合作餐廳、功能單一 |
| **探索功能** | 有限，以訂位為主 |

**用戶痛點**：
- 只能找到有合作的餐廳
- 探索功能薄弱

## 四、FeedNav 差異化定位

### 4.1 核心差異

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   Google Maps = 通用地圖 + 餐廳搜尋（附加功能）                  │
│   愛食記 = 美食文章聚合（內容平台）                              │
│   Foodpanda = 外送服務 + 餐廳探索（附加功能）                    │
│                                                                 │
│   FeedNav = 專為「正餐決策」設計的餐廳探索工具                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 獨特價值主張 (UVP)

| 競品解決的問題 | FeedNav 解決的問題 |
|---------------|-------------------|
| 「餐廳在哪裡？」 | 「今天吃什麼？」 |
| 「這家餐廳好不好？」 | 「我該去哪家？」 |
| 「怎麼訂外送？」 | 「收藏的餐廳怎麼管理？」 |

### 4.3 差異化功能

| 功能 | 差異化說明 |
|------|----------|
| **隨機推薦** | 解決選擇困難，競品都沒有 |
| **收藏地圖視覺化** | 讓收藏有價值感，Google Maps 做不到 |
| **情境標籤** | 聚餐/獨食/飲控，精準匹配需求 |
| **台北專精** | 深度資料 > 廣度覆蓋 |

## 五、競爭策略

### 5.1 不競爭的領域

| 領域 | 原因 |
|------|------|
| 用戶評論 | 愛食記、Google Maps 已經很強，且需要大量 UGC |
| 外送服務 | Foodpanda、UberEats 已經主導 |
| 訂位功能 | inline、EZTABLE 已經專精 |
| 全台覆蓋 | 先聚焦台北，深度優於廣度 |

### 5.2 競爭的領域

| 領域 | 策略 |
|------|------|
| 收藏管理 | 比 Google Maps 更好用（地圖視覺化） |
| 快速決策 | 比愛食記更快（不用看長文） |
| 情境探索 | 比所有競品更精準（情境標籤） |
| 選擇困難 | 獨家功能（隨機推薦） |

### 5.3 長期護城河

```
用戶投入（收藏、去過紀錄）
        ↓
累積個人化資料
        ↓
推薦越來越準
        ↓
轉換成本提高
        ↓
難以被取代
```

```

行動：使用 FeedNav 快速決策

```

## 八、總結

### FeedNav 定位

```
「專為正餐決策設計的台北餐廳探索工具」

不是地圖（Google Maps）
不是內容平台（愛食記）
不是外送（Foodpanda）
不是訂位（inline）

而是：幫你解決「今天吃什麼」的專屬工具
```

### 核心競爭力

1. **隨機推薦**：解決選擇困難（獨家）
2. **收藏地圖視覺化**：讓收藏有價值（差異化）
3. **情境標籤**：精準匹配需求（專業化）
4. **台北專精**：深度資料品質（聚焦）
