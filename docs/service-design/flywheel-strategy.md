# FeedNav 用戶參與飛輪效應策略

## 核心概念

飛輪效應的關鍵：**每個用戶行為都應該強化其他行為，形成自我增強的循環。**

---

## 一、平台現況分析

### 現有功能

| 功能 | 用途 | 用戶價值 |
|------|------|---------|
| 餐廳搜尋 | 找餐廳 | 解決「吃什麼」問題 |
| 地圖探索 | 視覺化瀏覽 | 地理位置探索 |
| 收藏功能 | 儲存喜歡的餐廳 | 個人口袋名單 |
| 造訪紀錄 | 記錄去過的餐廳 | 美食足跡 |
| 隨機推薦 | 解決選擇困難 | 驚喜感 |

### 核心挑戰

1. **搜尋多、收藏少**：用戶搜尋餐廳但不使用收藏功能
2. **一次性使用**：找到餐廳後就離開，不再回來
3. **缺乏習慣觸發**：沒有建立使用習慣的機制
4. **價值感知低**：用戶不覺得收藏比 Google Maps 好用

## 三、飛輪啟動策略

### Phase 1：建立核心循環（MVP）

```
目標：讓「探索 → 收藏 → 價值」循環運轉

關鍵行動：
1. 優化收藏體驗（一鍵收藏）
2. 展示收藏價值（地圖視覺化）
3. 突顯隨機功能（解決選擇困難）

成功指標：
- 每用戶平均收藏 10+ 家
- 7 日回訪率 > 25%
```

### Phase 2：強化價值感知

```
目標：讓用戶感受到累積的價值

關鍵行動：
1. 收藏里程碑通知
2. 個人化推薦（基於收藏）
3. 「附近的收藏」功能

成功指標：
- 收藏後查看率 > 30%
- 30 日回訪率 > 40%
```

### Phase 3：擴展飛輪

```
目標：加入社群元素強化飛輪

關鍵行動：
1. 收藏數量顯示（社會證明）
2. 分享功能優化
3. 朋友推薦功能

成功指標：
- 自發分享率 > 10%
- NPS > 50
```

## 四、用戶行為觸發設計

### 觸發點一覽

**核心場景：正餐決策（最難想的時刻）**

| 情境 | 觸發 | 期望行動 |
|------|------|---------|
| 正餐時間 | 「今天午餐/晚餐吃什麼？」 | 使用隨機推薦 |
| 聚餐規劃 | 「週末聚餐去哪？」 | 搜尋適合多人的餐廳 |
| 獨食情境 | 「一個人吃飯去哪好？」 | 篩選「一個人也適合」 |
| 飲控需求 | 「有什麼健康選項？」 | 篩選健康/低卡餐廳 |
| 發現新餐廳 | 「這家看起來不錯」 | 收藏餐廳 |
| 吃完餐廳 | 「這家去過了」 | 標記去過 |
| 朋友問推薦 | 「有什麼好吃的？」 | 分享收藏 |

### 習慣形成循環

```
外部觸發          內部觸發          行動          獎勵
   │                 │               │              │
   ▼                 ▼               ▼              ▼
通知/時間 ──▶ 「吃什麼？」 ──▶ 打開 FeedNav ──▶ 找到好餐廳
   │                 │               │              │
   └─────────────────┴───────────────┴──────────────┘
                     │
                     ▼
              習慣形成：想吃什麼 → 打開 FeedNav
```

## 五、數據指標追蹤

### 飛輪健康度指標

| 指標 | 定義 | 目標 |
|------|------|------|
| 收藏轉換率 | 訪問 → 收藏 | > 20% |
| 收藏累積率 | 每用戶每週新增收藏 | > 2 |
| 收藏使用率 | 從收藏頁進入餐廳 | > 30% |
| 隨機使用率 | 使用隨機功能比例 | > 15% |
| 回訪率 | 7 日內再次使用 | > 25% |

### 飛輪效應指標

```
飛輪加速度 = 收藏增長率 × 回訪率 × 使用深度

當飛輪加速度 > 1 時，飛輪正在加速
```

## 六、競爭優勢強化

### 對比 Google Maps

| 維度 | Google Maps | FeedNav |
|------|-------------|---------|
| 搜尋 | 通用搜尋 | 專為餐廳優化 |
| 篩選 | 有限篩選 | 料理類型、行政區 |
| 收藏 | 基本清單 | 地圖視覺化 |
| 隨機 | ❌ 無 | ✓ 解決選擇困難 |
| 台北專精 | 通用 | 台北深度資料 |

### 護城河建立

```
用戶投入越多 → 離開成本越高

投入類型：
1. 收藏資料（只存在於 FeedNav）
2. 造訪紀錄（個人美食歷史）
3. 使用習慣（熟悉的操作）
4. 累積價值（推薦越來越準）
```

## 九、總結

### 核心飛輪

```
探索 → 收藏 → 價值 → 回訪 → 探索更多
         ↑                    │
         └────────────────────┘
              自我增強
```

### 關鍵成功因素

1. **收藏體驗要比 Google Maps 好**：一鍵收藏、地圖視覺化
2. **隨機功能要有驚喜感**：解決選擇困難是獨特價值
3. **累積要有成就感**：里程碑、地圖覆蓋、美食足跡
4. **習慣要被培養**：午餐時間、週末出門想到 FeedNav

**飛輪一旦轉起來，就會越轉越快。**
