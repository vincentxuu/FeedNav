# 觸發點設計（Hook Design）

> **目標**：設計讓用戶養成使用習慣的觸發機制
> **框架**：基於 Nir Eyal 的 Hook Model（觸發 → 行動 → 變動獎勵 → 投入）

---

## 一、Hook Model 概述

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│     ┌──────────┐                      ┌──────────┐         │
│     │  觸發    │ ──────────────────▶ │  行動    │         │
│     │ Trigger  │                      │ Action   │         │
│     └──────────┘                      └────┬─────┘         │
│          ▲                                 │                │
│          │                                 ▼                │
│     ┌──────────┐                      ┌──────────┐         │
│     │  投入    │ ◀──────────────────  │  獎勵    │         │
│     │Investment│                      │ Reward   │         │
│     └──────────┘                      └──────────┘         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

| 階段 | 說明 | FeedNav 應用 |
|------|------|------------------|
| **觸發** | 驅動用戶採取行動的提示 | 通知、想吃什麼、週末出門 |
| **行動** | 用戶為了獲得獎勵而做的事 | 搜尋餐廳、收藏、隨機探索 |
| **獎勵** | 讓用戶滿足的回報（要有變動性） | 發現好餐廳、解決選擇困難 |
| **投入** | 用戶投入的時間/內容/資料 | 收藏清單、造訪紀錄、個人偏好 |

## 二、觸發類型設計

### 2.1 外部觸發（External Triggers）

由平台主動發送給用戶的提示

#### 類型 A：通知觸發（Push Notification）

```
┌─────────────────────────────────────────────────────────────┐
│  📱 通知類型設計                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【內容推薦類】每日/每週                                     │
│  ├─ 本週新開幕：「大安區新開了 3 家高評價餐廳」              │
│  ├─ 個人化推薦：「根據你的收藏，推薦這家日式料理」          │
│  └─ 熱門趨勢：「本週最多人收藏的餐廳是...」                 │
│                                                             │
│  【位置相關類】基於地點                                      │
│  ├─ 附近推薦：「你附近有 5 家你收藏的餐廳」                 │
│  └─ 新餐廳：「你常去的區域新開了一家店」                    │
│                                                             │
│  【行為提醒類】特定時機                                      │
│  ├─ 午餐時間：「今天中午吃什麼？試試隨機推薦」              │
│  └─ 週末提醒：「週末想去哪吃？看看你的收藏」                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**通知文案設計原則**：

| 原則 | 範例 |
|------|------|
| 具體化 | ❌「有新餐廳」→ ✅「大安區新開了麵屋一燈，評分 4.6」 |
| 製造好奇 | ❌「推薦餐廳」→ ✅「本週最多人收藏的是這家...」 |
| 行動導向 | ❌「歡迎回來」→ ✅「今天中午吃什麼？」 |
| 個人化 | ❌「熱門餐廳」→ ✅「根據你收藏的日式料理，推薦這家」 |

#### 類型 C：情境觸發

基於用戶所在情境的觸發

```
情境 1：正餐時間（最高價值時刻）
──────────────────
觸發時機：11:30-12:30（午餐）、17:30-18:30（晚餐）
觸發內容：「今天吃什麼？試試隨機推薦」
目標行動：使用隨機推薦或查看收藏

情境 2：聚餐規劃
──────────────────
觸發時機：週末前、特殊節日前
觸發內容：「週末聚餐去哪？看看適合多人的餐廳」
目標行動：搜尋聚餐適合的餐廳

情境 3：獨食情境
──────────────────
觸發時機：平日晚餐、週末一個人
觸發內容：「一個人吃飯？這些餐廳很適合」
目標行動：查看「一個人也適合」的餐廳

情境 4：飲控需求
──────────────────
觸發時機：週一、健身後
觸發內容：「想吃健康一點？看看這些選項」
目標行動：篩選健康/低卡餐廳

情境 5：到達新區域
──────────────────
觸發時機：位置變化
觸發內容：「你在信義區，這附近有 3 家你收藏的餐廳」
目標行動：查看附近收藏
```

### 2.2 內部觸發（Internal Triggers）

用戶自己產生的動機，不需要外部提示

```
┌─────────────────────────────────────────────────────────────┐
│  🧠 內部觸發培養                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  目標：讓用戶在特定情境自動想到 FeedNav                      │
│                                                             │
│  情境 → 內部觸發 → 行動                                      │
│                                                             │
│  「中午吃什麼？」                                            │
│   → 選擇困難、想找餐廳                                       │
│   → 打開 FeedNav 用隨機推薦                                  │
│                                                             │
│  「週末約朋友吃飯」                                          │
│   → 想找適合聚餐的餐廳                                       │
│   → 打開 FeedNav 搜尋                                        │
│                                                             │
│  「發現一家好餐廳」                                          │
│   → 想記下來下次再去                                         │
│   → 打開 FeedNav 收藏                                        │
│                                                             │
│  「去過一家餐廳」                                            │
│   → 想記錄已經去過                                           │
│   → 打開 FeedNav 標記去過                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**培養內部觸發的方法**：
1. 重複的外部觸發 → 形成習慣
2. 創造獨特價值（隨機推薦解決選擇困難）
3. 建立情感連結（我的口袋名單）

## 三、行動設計（降低摩擦）

### Fogg 行為模型

```
行動 = 動機 × 能力 × 觸發

當觸發出現時，用戶需要有足夠的動機和能力才會行動。
```

### 降低行動門檻

| 行動 | 現有摩擦 | 優化方案 |
|------|---------|---------|
| 搜尋餐廳 | 不知道搜什麼 | 提供熱門搜尋、快速篩選 |
| 收藏餐廳 | 覺得沒必要 | 一鍵收藏、展示收藏價值 |
| 隨機探索 | 不知道有這功能 | 首頁顯眼位置、午餐時推薦 |
| 標記去過 | 忘記去標 | 基於位置提醒、事後推送 |
| 分享餐廳 | 流程複雜 | 一鍵分享、自動生成預覽 |

### 快速動作設計

```
┌─────────────────────────────────────────────────────────────┐
│  @餐廳詳情頁                                                 │
│                                                             │
│  快速動作：                                                  │
│  ┌────────┬────────┬────────┬────────┐                     │
│  │ ❤️     │ ✓      │ 📍     │ 📤     │                     │
│  │ 收藏   │ 去過   │ 導航   │ 分享   │                     │
│  └────────┴────────┴────────┴────────┘                     │
│                                                             │
│  所有動作一鍵完成，不需要額外步驟                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 四、變動獎勵設計（Variable Reward）

### 為什麼需要「變動」？

固定獎勵會讓人麻木，變動獎勵讓人期待。

```
固定獎勵：每次搜尋都看到同樣的結果 → 很快無聊
變動獎勵：隨機推薦帶來驚喜 → 持續期待
```

### 三種獎勵類型

#### 類型 1：狩獵獎勵（Hunt）

搜尋和發現的樂趣

```
┌─────────────────────────────────────────────────────────────┐
│  狩獵獎勵設計                                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【發現驚喜】                                                │
│  • 隨機推薦功能 → 不知道會推薦什麼                          │
│  • 「你可能會喜歡」→ 意想不到的餐廳                          │
│  • 「隱藏版美食」→ 低調但高評價的店                          │
│                                                             │
│  【解決問題】                                                │
│  • 找到完美的約會餐廳                                        │
│  • 發現附近的好選擇                                          │
│  • 解決「今天吃什麼」的困擾                                  │
│                                                             │
│  【變動性】                                                  │
│  • 不知道會發現什麼餐廳                                      │
│  • 不知道隨機會推薦什麼                                      │
│  • 每次瀏覽都可能有新發現                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 類型 2：自我獎勵（Self）

個人成就和掌控感

```
┌─────────────────────────────────────────────────────────────┐
│  自我獎勵設計                                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【進度感】                                                  │
│  • 收藏數量累積：「你已經收藏了 50 家餐廳」                 │
│  • 造訪紀錄：「你已經探索了 30 家餐廳」                     │
│  • 區域覆蓋：「你已經收藏了 8 個行政區的餐廳」             │
│                                                             │
│  【成就感】                                                  │
│  • 收藏里程碑 → 「你是美食探險家！」                        │
│  • 造訪里程碑 → 「你已經吃遍大安區！」                      │
│  • 發現隱藏版 → 「你發現了一家隱藏版美食」                  │
│                                                             │
│  【掌控感】                                                  │
│  • 我的口袋名單（專屬於我的餐廳清單）                        │
│  • 我的美食地圖（視覺化的探索紀錄）                          │
│  • 個人偏好設定（推薦會更準確）                              │
│                                                             │
│  【變動性】                                                  │
│  • 不知道下一個里程碑是什麼                                  │

│  • 個人化推薦的準確度提升                                    │
```

## 五、投入設計（Investment）

### 投入創造價值

用戶投入越多，離開成本越高。

```
投入類型：

1. 資料投入
   └─ 收藏的餐廳、造訪紀錄、個人偏好
   └─ 這些資料「只存在於這裡」

2. 時間投入
   └─ 建立口袋名單的時間
   └─ 熟悉介面的學習成本

3. 情感投入
   └─ 「我的」口袋名單
   └─ 「我的」美食地圖

4. 價值累積
   └─ 收藏越多，推薦越準
   └─ 使用越久，資料越有價值
```

### 投入驅動下一次觸發

```
投入 → 儲存價值 → 創造觸發

例子：
用戶收藏了一家餐廳（投入）
→ 餐廳被加入收藏清單（儲存價值）
→ 「你收藏的餐廳附近有新店」（創造觸發）
→ 用戶回來探索（新的行動）
→ 循環繼續
```

## 六、習慣養成時程

### 21 天習慣養成計劃

```
第 1-3 天：體驗核心價值
──────────────────────
目標：讓用戶感受到 FeedNav 的價值
觸發：
• Day 1：歡迎 + 推薦附近餐廳
• Day 2：「今天中午吃什麼？試試隨機推薦」
• Day 3：「你收藏的餐廳附近有新選擇」

第 4-7 天：建立使用模式
──────────────────────
目標：培養「有需求就打開」的習慣
觸發：
• Day 4：午餐時間推薦
• Day 5：「週末去哪吃？」
• Day 6-7：基於位置的提醒

第 8-14 天：強化價值
──────────────────────
目標：展示累積的價值
觸發：
• 收藏里程碑通知
• 個人化推薦品質提升
• 「你的美食地圖越來越豐富了」

第 15-21 天：養成習慣
──────────────────────
目標：讓使用成為自然
觸發：
• 減少推送頻率
• 保持核心提醒（午餐、週末）
• 觀察主動回訪行為
```

## 七、通知系統規格

### 通知類型與頻率

| 類型 | 優先級 | 頻率限制 | 可關閉 |
|------|--------|---------|--------|
| 附近餐廳推薦 | 中 | 每日 1 次 | 是 |
| 午餐推薦 | 中 | 工作日 1 次 | 是 |
| 週末推薦 | 中 | 每週末 1 次 | 是 |
| 新餐廳通知 | 低 | 每週 1 次 | 是 |
| 收藏里程碑 | 低 | 達成時 | 否 |
| 週報 | 低 | 每週 1 次 | 是 |

### 通知渠道

| 渠道 | 使用時機 | 注意事項 |
|------|---------|---------|
| App Push | 高優先級、即時性 | 不可過於頻繁 |
| Email | 低優先級、彙整式 | 週報形式 |
| Web Push | 桌面用戶 | 取得許可 |
| In-App | 進入 App 時 | 非侵入式 |

### 通知文案模板

```
【午餐推薦】
標題：今天中午吃什麼？
內容：試試隨機推薦，讓 FeedNav 幫你決定 🎲

【週末推薦】
標題：週末去哪吃？
內容：你收藏的 3 家餐廳在附近，看看要去哪家？

【收藏里程碑】
標題：🎉 你已經收藏了 50 家餐廳！
內容：你是台北美食探險家！繼續探索更多美食 →

【週報】
標題：你的 FeedNav 週報
內容：本週 10 家新餐廳加入，你收藏的區域有 3 家 ✨
```

## 八、反模式警示

### 避免這些做法

| 反模式 | 問題 | 後果 |
|--------|------|------|
| 過度通知 | 每個小動作都發通知 | 用戶關閉通知 |
| 假緊急 | 「限時優惠」「即將結束」 | 失去信任 |
| 操控性文案 | 「你確定要離開嗎？」 | 反感 |
| 無用通知 | 「歡迎回來」 | 被忽略 |
| 不相關推薦 | 推薦用戶不感興趣的 | 降低信任 |

### 健康的觸發原則

```
✓ 觸發要有價值：推薦確實有用的餐廳
✓ 觸發要尊重用戶：提供關閉選項
✓ 觸發要誠實：不誇大、不製造假緊急
✓ 觸發要適量：寧缺勿濫
✓ 觸發要個人化：根據用戶偏好調整
```

## 九、實作優先序

### P0：基礎功能（MVP）

```
目標：先讓核心功能運作，通知系統精簡化

必做：
- [ ] 收藏功能完善
- [ ] 隨機推薦功能
- [ ] 收藏地圖視覺化
- [ ] 收藏里程碑通知（in-app，非 push）

不做：
- ✗ Push 通知（需要用戶許可，MVP 先跳過）
- ✗ Email 通知（需要 Email 服務整合）
- ✗ 午餐時間推薦（需要精準時機判斷）
```

### P1：基礎通知系統（Phase 2）

```
前提：MVP 核心功能已驗證有效

新增：
- [ ] 收藏里程碑 push 通知
- [ ] 週報 Email（每週日）
- [ ] 新餐廳通知（每週 1 次）
```

### P2：情境觸發（Phase 3）

```
前提：有足夠用戶數據

新增：
- [ ] 午餐時間推薦
- [ ] 週末探索提醒
- [ ] 基於位置的推薦
```

### P3：進階功能（Phase 4）

```
前提：有活躍用戶社群

新增：
- [ ] 基於收藏的個人化推薦
- [ ] 社群功能（朋友收藏）
- [ ] 智能推薦演算法
- [ ] 成就系統
```

### MVP 通知設計原則

```
1. 克制：寧缺勿濫，避免打擾用戶
2. 價值：每個通知都要有明確價值
3. 可關閉：用戶可以選擇不接收
4. 漸進：先驗證核心功能，再加通知
```
