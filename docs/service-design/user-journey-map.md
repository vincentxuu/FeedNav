# 用戶旅程地圖（User Journey Map）

> **目標**：完整描繪用戶從發現到成為活躍使用者的全旅程
> **背景**：主要流量來源為社群分享與搜尋，核心問題是「搜尋的人多，收藏的人少」

---

## 一、用戶角色定義（Persona）

### 主要角色：週末美食探索者

```
姓名：小美（代表 70% 的目標用戶）
年齡：28 歲
居住地：台北
探索頻率：每週 2-3 次外食

特徵：
- 有正職工作，外食是日常
- 有固定的朋友圈會一起吃飯
- 會看 IG/FB 美食帳號
- 在 Google Maps 標記過一些餐廳但沒整理
- 手機相簿有很多食物照片

心理狀態：
- 喜歡嘗試新餐廳但不知道去哪
- 覺得「記錄吃什麼」很麻煩
- 對美食推薦有興趣但懷疑真實性
- 喜歡看別人的推薦
```

### 次要角色

| 角色 | 特徵 | 佔比 |
|------|------|------|
| 飲控族群 | 注重熱量/健康、需要找適合的餐廳選項 | 15% |
| 聚餐規劃者 | 經常需要找適合多人聚餐、約會、商務的餐廳 | 10% |
| 獨食探索者 | 一個人吃飯、想找「一個人也適合」的餐廳 | 10% |
| 新台北人 | 剛搬來台北 < 1 年，積極探索城市 | 5% |

## 三、各階段詳細分析

### 階段 1：發現（Awareness）

**場景**：用戶在尋找正餐餐廳（最難決定的時刻）

```
情境舉例：
「今天中午/晚餐吃什麼？」（每日正餐決策）
「週末聚餐要去哪？」（多人聚餐）
「一個人吃飯，有什麼適合的？」（獨食情境）
「我在飲控，有什麼健康選項？」（飲控族群）
```

| 維度 | 內容 |
|------|------|
| **接觸點** | Google 搜尋、朋友推薦、社群分享 |
| **用戶行為** | 點擊連結、進入網站 |
| **用戶想法** | 「這個網站能幫我找到好餐廳嗎？」 |
| **情緒** | 😐 好奇但觀望 |
| **成功指標** | 連結點擊率（CTR） |

**設計機會**：
- 分享連結的預覽圖（OG Image）要吸引人
- 標題要清楚傳達價值「台北餐廳探索」
- SEO 優化讓搜尋能找到

**分享預覽優化**：
```
┌─────────────────────────────────────────┐
│  [餐廳精選照片]                          │
│                                         │
│  FeedNav                                │
│  「發現台北隱藏版美食」                   │
│  大安區日式料理推薦 — 4.5 星 · 中價位     │
└─────────────────────────────────────────┘
```

### 階段 2：探索（Exploration）

**場景**：用戶已經進入網站，開始瀏覽

| 維度 | 內容 |
|------|------|
| **接觸點** | 首頁、餐廳列表、地圖、篩選器 |
| **用戶行為** | 搜尋、篩選、瀏覽餐廳卡片、查看地圖 |
| **用戶想法** | 「這些餐廳看起來不錯」「篩選很方便」 |
| **情緒** | 🙂 有趣、想多看 |
| **成功指標** | 頁面停留時間、瀏覽餐廳數、跳出率 |

**用戶內心 OS**：
```
「這個介面很清楚」 → 繼續看
「可以按行政區和料理類型篩選」 → 產生信任
「評分和標籤一目瞭然」 → 覺得實用
「這家看起來不錯...」 → 想收藏
```

**關鍵設計**：
- 展示高品質的餐廳資訊
- 篩選功能要直覺好用
- 地圖瀏覽讓探索更有趣

**探索階段的引導**：
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │

│  ⭐ 4.6 · $$ · 捷運大安站 5 分鐘                             │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  ❤️ 156 人收藏                                               │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 想收藏這家餐廳？                                      │   │
│  │                                                      │   │
│  │ [ 登入收藏 ]  或  [ 繼續瀏覽 ]                       │   │
│  └─────────────────────────────────────────────────────┘   │
```

### 階段 3：註冊（Registration）

**場景**：用戶決定建立帳號

| 維度 | 內容 |
|------|------|
| **接觸點** | 註冊頁面、社群登入 |
| **用戶行為** | 填寫資料、社群登入 |
| **用戶想法** | 「希望不要太麻煩」「用 Google 登入最快」 |
| **情緒** | 😊 期待但有點煩 |
| **成功指標** | 註冊完成率、社群登入比例 |

**流失原因**：
- 步驟太多
- 要求太多資料
- 沒有社群登入選項

**優化建議**：
```
極簡註冊流程：

1. 選擇登入方式
   [ Google 登入 ] [ Discord 登入 ] [ Email 註冊 ]

2. Done! 開始探索你的專屬餐廳推薦
```

---

### 階段 4：首次使用（First Experience）⚠️ 關鍵階段

**場景**：剛註冊完，開始使用功能

| 維度 | 內容 |
|------|------|
| **接觸點** | 個人頁面、收藏功能、探索功能 |
| **用戶行為** | 收藏餐廳、設定偏好、瀏覽推薦 |
| **用戶想法** | 「收藏功能好用嗎？」「推薦準不準？」 |
| **情緒** | 🤔 猶豫、評估中 |
| **成功指標** | 首次收藏率、首日活躍率 |

**這是最大流失點！**

```
流失心理路徑：
註冊完 → 不知道要做什麼 → 覺得沒必要用 → 離開

期望心理路徑：
註冊完 → 看到推薦餐廳 → 收藏幾家 → 想繼續探索
```

**關鍵設計**（詳見 ux-first-mile.md）：
- 註冊後立即推薦符合位置的餐廳
- 一鍵收藏降低門檻
- 展示收藏的價值（可在地圖上看到所有收藏）

---

### 階段 5：回訪（Return）

**場景**：用戶離開後，什麼時候會再回來？

| 維度 | 內容 |
|------|------|
| **接觸點** | 通知、Email、搜尋 |
| **用戶行為** | 想找餐廳時打開、收到推薦時點進來 |
| **用戶想法** | 「上次收藏的那家在哪？」「有什麼新餐廳？」 |
| **情緒** | 😊 被記得、有幫助感 |
| **成功指標** | 7 日回訪率、30 日回訪率、DAU/MAU |

**回訪觸發點**：
```
被動回訪：
- 週末想找餐廳 → 打開 FeedNav
- 收到「本週新餐廳」通知 → 點進來看
- 朋友問推薦 → 打開收藏分享

主動回訪：
- 吃完一家想記錄「去過」
- 發現新區域想探索餐廳
- 想找特定類型的餐廳
```

**設計機會**：
- 通知系統（詳見 hook-design.md）
- Email 週報
- 「附近餐廳」推薦

---

### 階段 6：活躍使用（Engagement）

**場景**：用戶成為常態使用者

| 維度 | 內容 |
|------|------|
| **接觸點** | 所有功能 |
| **用戶行為** | 定期收藏、記錄造訪、探索新餐廳 |
| **用戶想法** | 「這是我的餐廳口袋名單」「下次吃什麼看 FeedNav」 |
| **情緒** | 🥳 依賴感、成就感 |
| **成功指標** | 收藏數、造訪紀錄數、活躍天數 |

**活躍關鍵**：
```
觸發點：
- 和朋友約吃飯（想找推薦）
- 發現一家新店（想收藏）
- 去過的餐廳想記錄評價

阻力：
- 功能太複雜
- 資料不準確
- 沒時間
```

**設計機會**：
- 降低記錄門檻（一鍵標記「去過」）
- 確保餐廳資料準確
- 提供價值讓用戶願意投入時間

```

  ↑
高 │                              🥳

   │        😊              ╭─╯
   │      ╭─╯ ╲           ╭─╯

   └──────────────────────────────────→ 時間

```

**情緒低點分析**：

| 低點 | 原因 | 對策 |
|------|------|------|
| 首次使用 | 不知道功能價值 | 立即展示推薦、引導收藏 |
| 找不到想要的 | 搜尋結果不佳 | 優化篩選、提供替代方案 |
| 資料不準 | 餐廳資訊過時 | 持續更新、用戶回報機制 |

## 五、關鍵時刻（Moments of Truth）

### MOT 1：第一次搜尋結果

```
成功：「這些餐廳看起來都不錯」→ 繼續探索
失敗：「搜尋結果跟我要的不一樣」→ 離開
```

**設計要點**：精準的搜尋與篩選

### MOT 2：註冊後看到的第一個畫面

```
成功：「推薦的餐廳就在我附近」→ 開始收藏
失敗：「都是我不感興趣的」→ 離開
```

**設計要點**：基於位置和偏好的個人化推薦

### MOT 3：第一次用收藏找餐廳

```
成功：「收藏真的很有用」→ 繼續使用
失敗：「找半天不如直接 Google」→ 放棄
```

**設計要點**：收藏功能要比 Google Maps 更好用

### MOT 4：隨機探索發現好餐廳

```
成功：「用隨機功能發現寶藏」→ 更常使用
失敗：「推薦的都不合我口味」→ 減少使用
```

**設計要點**：隨機功能要帶來驚喜感

## 八、衡量指標

### 漏斗指標

```
發現 → 探索：連結點擊率（目標 > 5%）
     │
     ▼
探索 → 註冊：訪客註冊轉換率（目標 > 10%）
     │
     ▼
註冊 → 首次使用：首次收藏率（目標 > 40%）
     │
     ▼
首次使用 → 回訪：7 日回訪率（目標 > 30%）
     │
     ▼
回訪 → 活躍：30 日活躍率（目標 > 20%）
```

### 體驗指標

| 指標 | 定義 | 目標 |
|------|------|------|
| NPS | 淨推薦分數 | > 50 |

| 首次價值時間 | 註冊到收藏第一家的時間 | < 2 分鐘 |
